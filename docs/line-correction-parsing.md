# LINE Text Correction Parsing

เอกสารนี้อธิบายรูปแบบข้อความที่ `line-relay` (`intake_server.py`) รับเป็นการแก้ไข draft
ระหว่างรอ `awaiting_approval` หรือในโหมด `awaiting_correction_input`

Implementation: `hermes/supervisor/services/intake_server.py`

## เข้าโหมดแก้ไข

| วิธี | ตัวอย่าง | ผลลัพธ์ |
| --- | --- | --- |
| ปุ่ม LINE | `แก้ไข` | `action=correct` → รอค่าที่จะแก้ |
| ข้อความ | `แก้ไข` | เข้าโหมดแก้ไข (ยังไม่มีค่า override) |
| ข้อความ | `แก้ไข 4=14` | แก้ทันทีโดยไม่ต้องเข้าโหมดก่อน |

ผู้ใช้ไม่จำเป็นต้องพิมพ์คำว่า `แก้ไข` นำหน้า หากข้อความตรงรูปแบบฟิลด์ด้านล่าง
ระบบจะถือเป็น `correction_command` โดยอัตโนมัติ

ปุ่ม **ยืนยัน** ยังกดได้ตลอด แม้ข้อมูล OCR จะมาไม่ครบในครั้งแรก

## ตัวคั่นค่า

รองรับทั้งสองทิศทาง:

- ชื่อฟิลด์แล้วตามด้วยค่า: `บัตรดี 850`, `บัตรดี=850`, `บัตรดี เป็น 850`
- ค่าแล้วตามด้วยชื่อฟิลด์: `850 บัตรดี`

ตัวคั่นที่รองรับ: `=`, `เป็น`, `คือ`, `ควรเป็น`, `:`

## สรุปบัตร (ballot summary)

| ฟิลด์ใน draft | ตัวอย่างที่รับ |
| --- | --- |
| `eligible_voters` (ผู้มีสิทธิ) | `ผู้มีสิทธิ 1000`, `มีสิทธิ 1000` |
| `voter_turnout` (ผู้มาใช้สิทธิ) | `ผู้มาใช้สิทธิ 900`, `ผู้ใช้สิทธิ 900`, `มาใช้สิทธิ 900` |
| `valid_ballots` (บัตรดี) | `บัตรดี 850`, `ดี 850`, `850 บัตรดี` |
| `invalid_ballots` (บัตรเสีย) | `บัตรเสีย 20`, `เสีย 20` |
| `vote_no` (ไม่ออกเสียง) | `ไม่ออกเสียง 30`, `ไม่ออกคะแนน 30`, `งดออกเสียง 30`, `vote no 30`, `no 30`, `งด 30` |

ข้อความเดียวสามารถมีหลายฟิลด์ได้ เช่น
`ผู้มีสิทธิ 1000 ผู้มาใช้สิทธิ 900 บัตรดี 850 บัตรเสีย 20 ไม่ออกเสียง 30`

### ไม่รองรับ

- การแมปเลขบรรทัดเอกสาร เช่น `1=1000 2=900 3=850` (บรรทัด 1–5)
- ต้องระบุชื่อฟิลด์หรือคำย่อ (`ดี`, `เสีย`, `no`, `งด`) แทน

## คะแนนผู้สมัคร

| รูปแบบ | ตัวอย่าง |
| --- | --- |
| เลขเบอร์ = คะแนน | `4=14` |
| มี prefix | `ผู้สมัคร 4=14`, `เบอร์ 4 เป็น 14`, `คะแนน 4=14` |

กฎสำคัญ:

- รูปแบบ `เลข=เลข` ต้องมีตัวคั่น `=` / `เป็น` / `คือ` / `ควรเป็น` เสมอ
- ข้อความสนทนาทั่วไป เช่น `เหมือนเบอร์ 4 จะหายไป` **ไม่** ถูกจับเป็นคำสั่งแก้ไข
- `ไม่ออกเสียง 30` ถูกจับเป็นสรุปบัตร (`vote_no`) ไม่ใช่คะแนนเบอร์ 3

## เขต (area)

| รูปแบบ | ตัวอย่าง |
| --- | --- |
| ชื่อฟิลด์ + เลขเขต | `เขต 13`, `แก้ไข เขต 13`, `area 13`, `เขต37`, `37 เขต`, `37เขต` |
| ชื่อฟิลด์ + ชื่อเขต | `เขต หนองจอก`, `เขตหนองจอก`, `แก้ไข เขต สายไหม`, `Ratchathewi` |
| เลข + คำว่าเขต | `13 เขต` |
| ชื่อเขตอย่างเดียว | `หนองจอก` (ต้องตรงกับ district master) |
| มีคำนำหน้า/คำลงท้าย | `'เขตราชเทวี'`, `เขต 37 กทม`, `ลำดับเขต 13` |

## ข้อความแนะนำอัตโนมัติ

เมื่อผู้ใช้ส่งข้อความที่ดูเหมือนต้องการแก้แต่ parse ไม่ได้ หรือส่งข้อความคลุมเครือระหว่างรอ approval
ระบบตอบด้วย `build_correction_guidance_text()` ซึ่งสรุปรูปแบบด้านบน

## LIFF form

ฟอร์ม LIFF (`/line/liff/correction`) ยังรองรับเฉพาะ **คะแนนผู้สมัคร** เท่านั้น
การแก้สรุปบัตรทำผ่านข้อความ LINE ตามเอกสารนี้

## Deploy

โค้ด parsing อยู่ใน image `line-relay` — หลังแก้ `intake_server.py` ต้อง rebuild container:

```bash
docker compose --env-file .env up -d --no-deps --build line-relay
```

Prod deploy script: [`deploy/ssm/deploy/intake-correction-parsing.json`](../../deploy/ssm/deploy/intake-correction-parsing.json)
